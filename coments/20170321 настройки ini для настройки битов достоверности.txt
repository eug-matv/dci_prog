В файле IKOPreSetup.ini есть опции в конце файла в группе [PROTOCOLS]. Там очень хорошо прокомменитировано.
Поясню, что для нас играют роль параметры, заканчивающиеся на ...ControlValueOfTrue. 

Например, если значение
; номер байта в поле данных пакета для получения значения достоверности номера  для RBS

; если равно 0, значение не анализируется
rbsNumberControlNByte = 12

; номер бита для  отвечающий за достоверность высоты
rbsNumberControlNBit = 7

; Значение бита определяющей достоверность 0 или 1
rbsNumberControlValueOfTrue = 0


Байт номер 12 (байты считаются от ) в информационной части пакета (или 14 байт если брать стартовые байты) у номера RBS, старший бит № 7 (биты начинаются от 0) - это бит контроля достоверности у номера борта RBS. 
Значение rbsNumberControlValueOfTrue = 0 указывает на то, что достоверным считается получение информации о номере RBS, 
если значение бита = 0. Если значение бита №7 байта № 12 равно 1, то информация о номере борта RBS НЕДОСТОВЕРНА.

Если значение rbsNumberControlValueOfTrue = 1, то достоверым будет значение номера борта при бите 7 в байте 12, равным 1, 
 а если этот бит будет равен 0, значет не достоверный номер борта.
